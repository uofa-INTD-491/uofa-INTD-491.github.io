## Problem

Makerspace staff were spending too much time clarifying incomplete fabrication requests. Students often submitted rough ideas without the details needed for scheduling, material planning, or safety review.

## Data

The assistant uses prior intake form submissions, common revision notes from staff, equipment capability summaries, and a simplified decision tree for materials and safety constraints.

## Method

The project turns the intake process into a guided conversation with progressive questions. Instead of one long form, it asks only for details that are relevant to the selected fabrication path.

## Results

Prototype testing suggested that students submitted more complete requests on the first try and staff needed fewer follow-up messages before scheduling fabrication support.

## How to run

1. Clone `https://github.com/example/makerspace-intake-assistant`.
2. Install the listed dependencies and seed the sample intake data.
3. Run the local application server and open the generated local URL.
