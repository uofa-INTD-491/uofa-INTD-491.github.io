## Problem

Raw sensor exports were difficult for non-specialists to interpret, especially when teams needed to explain findings to community partners in a short presentation window.

## Data

The dashboard uses exported sensor readings for turbidity, pH, conductivity, and temperature, along with sampling-site metadata that supports site-to-site comparisons and recent-trend summaries.

## Method

The team transformed raw time-series files into a presentation-oriented dashboard with small multiples, anomaly flags, and site comparison views. The design favors quick interpretation over exploratory analysis.

## Results

RiverWatch made it easier for students to explain sensor patterns to non-technical audiences, especially when highlighting recent anomalies and comparing stream conditions between sites.

## How to run

1. Clone `https://github.com/example/riverwatch-sensor-dashboard`.
2. Install the project dependencies and load the sample sensor export files.
3. Start the dashboard locally and open the provided URL in a browser.
