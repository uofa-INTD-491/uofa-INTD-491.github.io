## Problem

Transit accessibility issues were being discussed through fragmented screenshots, notes, and route tables. The team wanted a format that could communicate rider experience more clearly during stakeholder review sessions.

## Data

The storyboard uses route schedules, stop amenities, sidewalk observations, elevator outage notes, and short ride-audit summaries collected during field visits.

## Method

The team organized each trip as a visual sequence with a few high-value state changes: departure, transfers, obstacles, and recovery options. The interface is optimized for quick narrative walkthroughs rather than deep schedule search.

## Results

Review meetings moved faster because the storyboard gave participants a shared, concrete picture of where accessibility friction accumulated along a trip.

## How to run

1. Clone `https://github.com/example/transit-access-storyboard`.
2. Install dependencies and load the sample route package included in the repository.
3. Start the local preview server and open the storyboard in a browser.
