## Problem

Students reported that existing booking tools showed room availability, but not the context needed to decide quickly which space actually fit their group. The team focused on shortening that decision loop.

## Data

The project combines campus room metadata, accessibility notes, building floor plans, and manually reviewed pod attributes such as seat count, display support, and quiet-zone status.

## Method

The team designed a searchable interface that pairs a simplified map with a small set of high-value filters. The card layout was shaped by user interviews and quick hallway usability tests focused on how students pick a room under time pressure.

## Results

Students could identify a suitable study pod faster than with the original booking flow because the prototype reduced tab-switching and surfaced accessibility constraints early in the search process.

## How to run

1. Clone the repository from `https://github.com/example/campus-study-pod-finder`.
2. Install the dependencies listed in that repository.
3. Start the local development server and open the project in a browser.
